Getting Started

Welcome to the sayIt Playground API documentation. This guide will help you seamlessly integrate fun games and trivia into your applications. Our API uses JSON for data exchange, making it easy and efficient to add engaging features to your projects.

The Playground API provides a wide variety of interactive games, trivia questions, and entertainment endpoints, allowing you to quickly add playful and engaging experiences to your websites and apps with simple HTTP requests.

Base URL

All API requests should be made to the following base URL: 

http://127.0.0.1:5000/api/v1/

All API responses are returned in JSON format with UTF-8 encoding.

Authentication

Authentication is handled with a simple API key. You can get one here:

Get API Key

API Key Authentication

Via Querystring 
GET https://127.0.0.1:5000/api/v1/all-game-types?api_key={{api_key}}
Via X-Api-Key HTTP header 
X-Api-Key: {{api_key}}
Via Authorization HTTP header 
Authorization: {{api_key}}

An Via the Authorization HTTP header. Including Bearer is optional, and be sure not to base 64 encode it like you may have seen in other authentication tutorials

If you don't append your API key correctly, or your API key is invalid, you will receive a 401 - Unauthorized HTTP error.

ðŸŽ® Playground API Endpoints

Playground API has three endpoints:

GET /api/v1/all-game-types

Returns a list of all available game types (e.g., Would You Rather, Never Have I Ever, Trivia, Did You Know, etc.) currently supported by the API.

GET /api/v1/get-random?game_type=...

Returns a single random game from the type specified in the query (e.g., one Trivia or one Would You Rather question).

GET /api/v1/get-by-type?game_type=...

Returns a list of games of the specified type. Useful when you want to fetch multiple entries from a specific category.

)
print(response.json())
